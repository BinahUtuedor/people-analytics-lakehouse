### Schema Responsibilities

| Schema | Purpose |
|----------|----------|
| **public** | Stores the operational HR information system containing transactional workforce data generated by the simulator. |
| **catalogue** | Stores enterprise metadata including dataset registration, column definitions, lineage, glossary terms, ownership and data quality metrics. |
| **data_sharing** | Stores governance information for external data sharing including API consumers, authentication, access policies, audit logs and export requests. |

# Project Design Principles

The platform has been designed using a number of enterprise software engineering principles.

## Separation of Concerns

Each module has a single, clearly defined responsibility. Operational systems, ETL, analytics, metadata, APIs and governance are implemented independently to improve maintainability and reduce coupling.

## Layered Architecture

The project follows a layered architecture from operational data generation through data engineering, analytics and business consumption.

## Metadata-Driven Design

Business metadata, technical metadata, lineage, ownership and classifications are stored independently from operational data, enabling automated documentation, governance and data catalogue generation.

## Reusable Reference Data

Static organisational reference data such as departments, locations and leave types are maintained centrally and reused throughout the simulator, database seeding, validation framework and reporting layers.

## Security by Design

The platform separates operational data from external data-sharing services. Authentication, authorisation, API governance and audit logging are implemented independently to support secure access to approved data products.

## Data Governance

Metadata management, quality validation, lineage tracking and catalogue registration are treated as first-class capabilities rather than afterthoughts, reflecting modern enterprise data governance practices.

## Scalability

The platform is designed to evolve from local development to cloud deployment through modular ETL pipelines, Spark processing, dbt transformations and containerised services.

## Testability

Each major component includes dedicated test suites, enabling automated validation of business logic, ETL processes, APIs, metadata and analytics models through continuous integration workflows.
